# تقارير الجودة والتوثيق

هذا المجلد يحتوي على تقارير جودة الكود والتوثيق المُولدة آلياً.

## الملفات المتاحة

### تقارير التوثيق
- `doc-coverage.json` - تقرير تغطية التوثيق (TSDoc)
- `type-coverage.txt` - تقرير تغطية الأنواع (TypeScript)

### تقارير البنية
- `repo-map.json` - خريطة المستودع والوحدات
- `module-graph.svg` - مخطط العلاقات بين الوحدات

### تقارير الجودة
- `link-check.txt` - تقرير فحص الروابط
- `api-report.json` - تقرير API (يُولد بواسطة API Extractor)

## كيفية قراءة التقارير

### تقرير تغطية التوثيق
```bash
# عرض التقرير
cat .reports/doc-coverage.json | jq '.'

# فحص التغطية الإجمالية
cat .reports/doc-coverage.json | jq '.overall.coverage'
```

### تقرير تغطية الأنواع
```bash
# عرض التقرير
cat .reports/type-coverage.txt
```

### خريطة المستودع
```bash
# عرض خريطة المستودع
cat .reports/repo-map.json | jq '.'

# عرض الوحدات الحرجة
cat .reports/repo-map.json | jq '.criticalModules'
```

## توليد التقارير

### تشغيل جميع التقارير
```bash
npm run ci:docs
```

### تشغيل تقارير محددة
```bash
# تقرير تغطية التوثيق
npm run doc:coverage

# تقرير تغطية الأنواع
npm run type:coverage

# فحص الروابط
npm run linkcheck

# بناء التوثيق
npm run doc:build
```

## تحديث التقارير

التقارير تُحدث تلقائياً عند:
- دفع تغييرات إلى الفرع الرئيسي
- إنشاء Pull Request
- تشغيل سير عمل CI/CD

## استكشاف الأخطاء

### تقرير تغطية منخفض
1. راجع الملفات المذكورة في التقرير
2. أضف توثيق TSDoc للعناصر غير الموثقة
3. أعد تشغيل `npm run doc:coverage`

### روابط معطلة
1. راجع `link-check.txt`
2. أصلح الروابط المعطلة
3. أعد تشغيل `npm run linkcheck`

### أخطاء في الأنواع
1. راجع `type-coverage.txt`
2. أضف أنواع TypeScript صريحة
3. أعد تشغيل `npm run type:coverage`

## المساهمة

للمساهمة في تحسين الجودة:
1. راجع التقارير بانتظام
2. أصلح المشاكل المذكورة
3. أعد تشغيل التقارير للتأكد من الإصلاح
4. أرسل Pull Request مع التحسينات
