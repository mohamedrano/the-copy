# تقرير شامل عن مجلد `apps` - مشروع The Copy

## 📋 نظرة عامة

مجلد `apps` يحتوي على **5 تطبيقات رئيسية** متخصصة في تحليل النصوص الدرامية والسيناريوهات العربية، تعمل ضمن نظام monorepo متكامل باستخدام pnpm workspace. كل تطبيق مستقل تقنياً ولكنه يتكامل مع الآخرين من خلال التطبيق الرئيسي `the-copy`.

---

## 🏗️ البنية المعمارية

### النظام العام
- **نوع المشروع**: Monorepo باستخدام pnpm workspaces
- **إدارة التبعيات**: pnpm مع overrides موحدة لـ React
- **اللغة الأساسية**: TypeScript (strict mode)
- **البناء**: Vite لكل تطبيق
- **التشغيل**: Docker-ready مع docker-compose

### التطبيق الرئيسي - Shell Application
**`the-copy`** هو التطبيق الموحد الذي يجمع جميع التطبيقات الأخرى في واجهة واحدة:
- **الغرض**: لوحة تحكم موحدة تعرض جميع التطبيقات
- **التقنيات**: React 19 + TypeScript + Vite
- **المميزات**: 
  - مراقبة الحالة الصحية للتطبيقات
  - تحميل التطبيقات عبر iframe
  - دعم RTL كامل للعربية

## 📱 التطبيقات الخمسة

**الغرض**: محرر نصوص متخصص للسيناريوهات العربية

**المواصفات التقنية**:
- **الحجم**: 204 KB (59.49 KB مضغوط)
- **البورت**: 5178
- **التبعيات الأساسية**: React 19, Lucide React
- **المميزات**:
  - دعم كامل للغة العربية (RTL)
  - واجهة بسيطة وسهلة الاستخدام

**البنية**:
```
src/

├── main.tsx                  # نقطة الدخول
├── ScreenplayEditor.tsx      # المحرر الأساسي
```

### 2. محلل الدراما (`drama-analyst`)
**الغرض**: نظام ذكاء اصطناعي متقدم لتحليل النصوص الدرامية

**المواصفات التقنية**:
- **البورت**: 5179
- **الوكلاء**: 29 وكيل AI متخصص
- **API**: Google Gemini 1.5 Flash/Pro
- **التبعيات الرئيسية**: 
  - `@google/generative-ai`
  - `mammoth` (معالجة مستندات Word)
  - `react-dropzone`

**المميزات**:
- تحليل شامل للنصوص الدرامية
- دعم ملفات Word (.docx)
- نظام وكلاء متخصصين

- اختبارات شاملة (Unit + E2E)

**البنية**:
```
├── agents/                   # 57 ملف للوكلاء المتخصصين
├── services/                 # 17 خدمة

├── core/                     # النواة الأساسية
├── orchestration/            # تنسيق العمليات

```

### 3. المحطات التحليلية (`stations`)
**الغرض**: نظام تحليل متقدم للنصوص الدرامية عبر 7 محطات متسلسلة

**المواصفات التقنية**:
- **البورت**: 5000 (خادم) + 3000 (واجهة)
- **قاعدة البيانات**: PostgreSQL + Drizzle ORM
- **التخزين المؤقت**: Redis
- **الواجهة**: React + shadcn/ui + Tailwind CSS
- **API**: Express + TypeScript

**المحطات السبع**:
1. **المحطة 1**: التحليل الأساسي (الشخصيات والعلاقات)
2. **المحطة 2**: التحليل المفاهيمي (الثيمات والنوع)
3. **المحطة 3**: بناء شبكة الصراع (الأهم)
4. **المحطة 4**: قياس الكفاءة والفعالية
5. **المحطة 5**: التحليل الديناميكي والرمزي
6. **المحطة 6**: التشخيص والعلاج
7. **المحطة 7**: التصور والعرض النهائي

**المميزات**:
- خط أنابيب تحليلي متسلسل
- تصورات تفاعلية للشبكات
- مقاييس كمية للجودة الدرامية
- نظام تشخيص ذكي

### 4. العصف الذهني متعدد الوكلاء (`multi-agent-story`)
**الغرض**: منصة تطوير قصصية تعاونية باستخدام 11 وكيل AI

**المواصفات التقنية**:
- **البورت**: 5181
- **الوكلاء**: 11 وكيل متخصص
- **الخادم**: FastAPI (Python)
- **قاعدة البيانات**: PostgreSQL + Prisma
- **التخزين المؤقت**: Redis
- **التواصل**: WebSockets

**عملية التطوير (5 مراحل)**:
1. **الملخص الإبداعي**: تحديد الفكرة الأولية
2. **توليد الأفكار**: إنشاء فكرتين متنافستين
3. **المراجعة المستقلة**: تقييم كل وكيل لكل فكرة
4. **المناقشة التنافسية**: مناقشة الوكلاء للدفاع عن أفكارهم
5. **القرار النهائي**: تقرير نهائي يحدد الفكرة الفائزة

**المميزات**:
- إدارة جلسات متكاملة
- تحديثات في الوقت الفعلي
- بنية جاهزة للإنتاج مع Docker

### 5. التطبيق الموحد (`the-copy`)
**الغرض**: التطبيق الرئيسي الذي يجمع جميع التطبيقات

**المواصفات التقنية**:

# تقرير شامل عن مجلدات المشروع

## 📁 1. مجلد `packages/`

### الوصف
مجلد يحتوي على **حزم مشتركة** (shared packages) مصممة للاستخدام عبر المشروع. يتضمن ثلاثة حزم أساسية:

#### الحزم الفرعية:
- **`shared-types/`**: حزمة الأنواع المشتركة
  - يحتوي على `index.ts` للأنواع العامة
  - إعداد TypeScript منفصل
  - نسخة 1.0.0

- **`shared-ui/`**: حزمة واجهة المستخدم المشتركة
  - يعتمد على React 19 و Lucide React للأيقونات
  - يحتوي على مكونات UI قابلة لإعادة الاستخدام
  - نسخة 1.0.0

- **`shared-utils/`**: حزمة الأدوات المساعدة
  - يحتوي على وظائف مساعدة مشتركة
  - إعداد TypeScript منفصل
  - نسخة 1.0.0

### الهدف
تنظيم الكود المشترك وتجنب التكرار عبر المشروع باستخدام بنية monorepo.

- **`common/`**: مكونات مشتركة

- مكونات الصفحات: `HomePage`, `ProjectsPage`, `TemplatesPage`, `ExportPage`, `SettingsPage`

#### 8.3 مجلد `config/` (5 ملفات):
- إعدادات الوكلاء والبيئة والنصوص التوجيهية

#### 8.4 مجلد `lib/` (4 ملفات):
- مكتبات الذكاء الاصطناعي وأنواع Gemini
